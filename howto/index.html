Android Testing

A How-To Guide

This guide will show you how to test an Android application using the JUnit, Espresso and Robolectric API's

Why did I write this how-to guide?

I wrote this guide for a couple of important reasons.

While the Android Developer and Android Testing Support library websites provide some documentation for setup and testing concepts, they neglect to provide support on how to deal with various issues with dependencies and versions that arise during installation, even if their setup instructions are followed exactly. This is why I found the current official documentation lacking in terms of troubleshooting during set up of the testing environment, which can lead to frustration, especially if the developer is eager to get up and running and start writing tests for their application. It can be annoying for developers when they find only minimal troubleshooting information and sometimes obscure error messages. Because good documentation does not currently exist for helping a developer deal with these issues to get up and running, I wrote this guide to hopefully fill that gap.

I have found that there is little guidance provided for beginners to Android development and testing. Much of the existing documentation assumes working knowledge of how to write tests for Android applications. This is a great opportunity to review the basics of writing unit tests, and provide some exposure to automated UI and integration testing. Thus, another goal of this guide is to help developers quickly get started on writing quality tests for their Android apps.

Features

This is a how-to guide on testing Android applications. It includes steps on the installation, writing, and execution of tests using:
  • JUnit API for local and instrumented unit testing
  • Espresso API for automated integration testing of the UI
  • Robolectric API for automated integration testing of the UI

Documentation

Where appropriate, this guide will link to official documentation to provide background information.

Requirements

This guide assumes basic knowledge of native Android development in Android Studio, but assumes no experience in setting up JUnit and Espresso in Android Studio, or testing an Android application.
